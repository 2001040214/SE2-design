Question 1
The most reliable defense against SQL injection is…
  • A. Escaping user input manually everywhere
  • B. Parameterized queries / prepared statements
  • C. Using GET instead of POST
  • D. Turning off database logs
Hint: Separate code from data.
Show answer
Correct: B. Parameterized queries / prepared statements
Explanation: Separate code from data.
Question 2
Command injection often happens when an app…
  • A. Uses HTTPS
  • B. Builds shell commands by concatenating untrusted input
  • C. Hashes passwords
  • D. Uses CORS
Hint: Shell + user input.
Show answer
Correct: B. Builds shell commands by concatenating untrusted input
Explanation: Shell + user input.
Question 3
A strong general strategy to reduce injection risk is…
  • A. Block all special characters always
  • B. Use allowlists and safe APIs, validate at boundaries
  • C. Only validate in the browser
  • D. Hide errors from users
Hint: Allowlist + safe sinks.
Show answer
Correct: B. Use allowlists and safe APIs, validate at boundaries
Explanation: Allowlist + safe sinks.
Question 4
Template injection becomes possible when…
  • A. Templates are stored in a database
  • B. User-controlled data is treated as template code/expression
  • C. Cookies are HttpOnly
  • D. TLS is enabled
Hint: Data executed as code.
Show answer
Correct: B. User-controlled data is treated as template code/expression
Explanation: Data executed as code.
